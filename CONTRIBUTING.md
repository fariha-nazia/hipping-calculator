# Contributing Guidelines

Thanks for your interest in contributing to this project!

## How to contribute

1. **Fork** this repository to your GitHub account.
2. **Clone** your fork to your local machine.
3. Create a **new branch** for your change:
   - Example: `feature/add-shipping-rule` or `fix/bug-in-calculation`
4. Make your changes and **test** them.
5. **Commit** your changes with a clear message (see below).
6. **Push** your branch to your fork.
7. Open a **Pull Request (PR)** to the main repository.

## Branch naming

Use one of these formats:
- `feature/short-description`
- `fix/short-description`
- `docs/short-description`

## Commit message style

Use short and clear commit messages, for example:
- `Add code of conduct`
- `Fix shipping calculation for large orders`
- `Update README with setup steps`

## Pull Request rules

- Keep PRs focused (one change per PR).
- Add a clear PR title and description.
- Make sure the code runs without errors.
- If your PR fixes a bug, explain how to reproduce the bug and how you fixed it.
